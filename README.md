# 캠핑캐치 (CampCatch) — 김포 3시간 오토캠핑 대시보드

김포 집 기준 **약 3시간 이내** 지역의 **오토캠핑장**을, 조회일의 **계절**과 **거리**로
추천 순위를 매겨 보여주는 대시보드입니다. **실데이터** 기반입니다.

- **데이터 출처**: 고캠핑(한국관광공사) 공개 API — 이용허락범위 제한 없음
- **수록**: 김포 3시간 이내 자동차야영장(오토캠핑) **143곳** (파주·연천·포천·강원)

## 실데이터로 되는 것 / 예약처에서 확인할 것

| 항목 | 상태 |
|---|---|
| 캠핑장명·지역·주소·오토 규모 | ✅ 고캠핑 실데이터 |
| 김포 거리·예상 이동시간(좌표 계산)·3시간 필터 | ✅ |
| 계절 추천 (여름 물놀이·계곡 / 겨울 온수·눈 등) | ✅ 테마환경 태그 기반 |
| **날씨·기온·강수확률·바람(풍속/돌풍)** | ✅ Open-Meteo (D-16 이내만) |
| 시설·환경 태그, 대표 사진, 예약처 링크 | ✅ |
| **가격 · 실시간 자리 · 후기** | ❌ API 미제공 → 예약처/네이버로 연결 |

> 후기 데이터 실제 연동과 실시간 자리 조회(“실시간” 배지)는 백엔드가 필요한 **2단계 과제**입니다.

## 캠핏과 다른 점 — "지금 갈까? 어디로?" 결정 도구

단순 추천을 넘어, 김포 3시간·오토·가족으로 이미 좁혀진 **내 후보**를 놓고 바로 출동을 결정하게 돕습니다.

- **날씨·바람 결합** (Open-Meteo, 무키·CORS) — 143곳을 한 번에 조회.
  캠핑엔 바람이 가장 치명적이라 **풍속을 1급 지표**로 두고 강풍은 크게 감점.
  예보가 믿을 만한 **D-16 이내에서만** 정렬에 반영하고, 그 밖은 억지로 쓰지 않고 계절 특성으로 추천.
- **빠른 날짜** — `[내일] [이번 주말] [다음 주말]` 원탭 (기본값 = 이번 주말).
- **자리 확인 추적** — 각 카드에서 `있음/없음/예약`을 찍으면 localStorage에 기억.
  다음에 열어도 "어제 자리없음"이 남아 재확인을 줄이고, 상태별로 정렬(자리있음↑·자리없음↓)·강조.

## 기능

- **추천순 랭킹** — 계절 적합 + 시설 충실도 + 오토 규모 + 아이 좋은 곳 + 온라인예약 + (D-16 날씨) − 거리
- **계절 추천 큐레이션** — 조회일 계절에 맞는 곳을 상단에 우선 노출
- **선호 우선** — ☆ 지정 캠핑장을 최상단 고정 (localStorage 저장)
- **아이 좋은 곳 배지** — 놀이터·물놀이장·계곡 등 보유 시 표시
- 상세: 사진 · 시설 태그 · 날씨/바람 · **예약처 이동 / 네이버 후기 / 김포에서 길찾기(카카오맵)**
- 라이트/다크 테마 자동, 모바일 우선 반응형

## 조건 (기본값)

- 지역: 파주 · 연천 · 포천 · 강원 (김포 집 기준 약 3시간 이내)
- 유형: 오토캠핑

## 파일 구조

```
camping-dashboard/
├─ index.html
├─ css/styles.css
├─ js/
│  ├─ config.js     # 계절·지역 설정 (수기)
│  ├─ sites.js      # 캠핑장 실데이터 (자동 생성)
│  ├─ links.js      # 죽은 예약/홈 링크 목록 (자동 생성, 검증기 산출물)
│  ├─ logic.js      # 랭킹·계절·날씨·링크 로직
│  └─ app.js        # 상태·렌더링·이벤트·날씨 조회
└─ tools/
   ├─ build_data.py   # 고캠핑 API → sites.js 생성기
   └─ check_links.py  # 예약/홈 링크 검증 → links.js 생성 (게시 전 실행)
```

## 게시 전 링크 검증 (권장)

예약처 링크는 폐업·주소변경으로 죽는 일이 잦습니다. 게시 전 아래를 돌리면
**확실히 죽은 링크(404·DNS 실패·오타 도메인 등)만** 골라 `js/links.js`에 기록하고,
앱은 그 링크를 **네이버 검색으로 자동 폴백**해 404를 막습니다(온라인예약 배지·상단 부스트에서도 제외).

```bash
python3 tools/check_links.py          # 리포트만 (지금 상태 점검)
python3 tools/check_links.py --emit   # 리포트 + js/links.js 재생성
```

브라우저와 동일하게 판정합니다 — URL 뒤 탭/공백 제거, 한글 도메인 IDNA 인코딩,
봇차단(401/403/405)은 '살아있음'. 살아있는 링크를 실수로 죽이지 않도록 **보수적**으로 판정합니다.

## 데이터 갱신 (실데이터 재생성)

고캠핑 API 키는 **환경변수로만** 사용하며 저장소에 커밋하지 않습니다.

```bash
GOCAMPING_KEY="발급받은_일반인증키" python3 tools/build_data.py
# → js/sites.js 재생성 (김포 3시간 이내 오토캠핑장)
```

API 키 발급: [공공데이터포털](https://www.data.go.kr) → “한국관광공사_고캠핑 정보 조회서비스” 활용신청.

## 실행 / 배포

빌드 과정 없음. `index.html`을 브라우저로 열거나 로컬 서버(`python3 -m http.server`)로 확인.
GitHub Pages 정적 호스팅으로 배포합니다.
